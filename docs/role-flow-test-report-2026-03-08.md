## 三、這次修正的關鍵點

### `ADM-03` 修正內容
原本 smoke 腳本在 `ADM-03` 直接尋找「轉為追蹤」按鈕，與目前產品狀態機不一致。

目前產品正確流程是：
1. 填報者回填後，案件狀態為「已提案」
2. 管理者先按「進入審核」
3. 進入「審核中」後，管理者才看得到「轉為追蹤」
4. 填報者送出追蹤提報
5. 管理者審核後結案或繼續追蹤

已修改 `scripts/role-flow-smoke.cjs`：
- `ADM-03` 先執行 `進入審核`
- 再執行 `轉為追蹤`
- 並補上 `pendingTracking` 仍應為空的驗證

## 四、管理者 / 填報者流程結論

### 1. 管理者開單
- 通過。
- 可登入、進入 `#create`、指定處理單位與處理人員、建立新矯正單。
- 焦點流程 `ADMIN-CREATE` 通過。

### 2. 填報者回填矯正單
- 通過。
- 可進入回填頁，完成矯正措施、根因分析與根因消除措施送出。
- 焦點流程 `REPORTER-RESPOND` 通過。

### 3. 管理者轉追蹤
- 通過。
- 管理者可先進入審核，再轉為追蹤。
- 焦點流程 `ADMIN-TO-TRACKING` 通過。

### 4. 填報者送追蹤提報
- 通過。
- 可進入 `#tracking/<id>`，送出追蹤內容。
- 若選擇擬請結案，必須附佐證檔。
- 焦點流程 `REPORTER-TRACKING` 通過。

### 5. 管理者審核追蹤並結案
- 通過。
- 送出後會產生 `pendingTracking`
- 管理者審核後可清除 `pendingTracking`，寫入追蹤紀錄並結案。
- 焦點流程 `ADMIN-CLOSE` 通過。

### 6. 檢核表 / 教育訓練填報
- 通過。
- 檢核表草稿、正式送出、教育訓練草稿、教育訓練正式送出皆可正常完成。
