# 正式上線前測試清單

- 版本日期：2026-03-08
- 適用對象：`C:/AI/ISMS-Form-Redesign`
- 使用時機：準備對外 Demo、校內試營運、正式上線前驗收

## 一、環境確認

1. 本機或測試主機可正常開啟首頁
2. `http://127.0.0.1:8080/` 或正式網址回應 `200`
3. 瀏覽器 console 無未處理 error
4. 測試帳號可正常登入
5. 測試資料可重置或可隔離

## 二、帳號與權限驗收

1. `admin` 可見管理頁與完整功能
2. `unit1` 無法開單、無法進入管理頁
3. `user1` 可檢視同單位案件
4. `viewer1` 可唯讀檢視，但不可填報
5. 未授權角色不能直接透過 hash 進入受限頁面

## 三、矯正單主流程

1. 管理者可建立新矯正單
2. 開單後可正確指派處理單位與處理人員
3. 填報者可回填矯正措施
4. 回填後資料會正確保存到案件明細
5. 管理者可進入審核
6. 管理者可轉為追蹤
7. 填報者可送出追蹤提報
8. 若選擇擬請結案，系統會要求佐證附件
9. 管理者可審核追蹤並結案
10. 結案後案件不可再編輯

## 四、檢核表流程

1. 可建立草稿
2. 草稿儲存後仍留在編輯頁
3. 正式送出後導向明細頁
4. 送出後不可再當草稿編輯
5. 統計列表可看到最新狀態

## 五、教育訓練流程

1. 可建立草稿
2. 草稿儲存後仍留在編輯頁
3. 正式送出需帶簽核附件
4. 送出後導向明細頁
5. 管理者可在統計頁看到最新資料

## 六、資料與附件驗收

1. 開單、回填、檢核表、教育訓練的關鍵欄位都有存檔
2. 附件上傳後可在明細頁看到
3. 同一案件的歷程紀錄有正確追加
4. 測試資料不會污染正式資料
5. 日期、單號、狀態顯示一致

## 七、自動化測試驗收

1. `npm run test:role:permission` 通過
2. `npm run test:role:probe` 通過
3. `npm run test:role:focus` 通過
4. `npm run test:role:smoke` 通過
5. `npm run test:role:all` 通過
6. 所有新產物都落在帶日期的 `test-artifacts/` 目錄

## 八、上線前最後人工檢查

1. 首頁、側欄、主要卡片無版面破版
2. 手機版至少可完成登入與查詢
3. 主要表單必填提示清楚
4. 權限不足時導回頁面與文案一致
5. 報表、統計頁文字無亂碼
6. Logo、favicon、靜態資源無 404

## 九、建議放行標準

- P0：主流程、權限、自動化測試全部通過
- P1：版面與文案問題不影響流程，可列入上線後修正
- P2：純視覺優化不阻擋上線

## 十、建議驗收紀錄方式

1. 記錄測試日期與 commit hash
2. 保存 `test-artifacts/` 產物
3. 若有人工測試，補上測試人與結果
4. 上線前最後一輪至少保留一份完整報告
