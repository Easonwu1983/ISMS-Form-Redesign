# 內部稽核管考追蹤系統操作手冊（新手版）

## 1. 這份手冊要解決什麼
這份手冊給第一次使用系統的人，目標是讓你可以自己完成：
- 登入與密碼重設
- 矯正單完整流程（開單、回填、審核、追蹤、結案）
- 內稽檢核表填報與草稿管理
- 教育訓練時數統計（名單、暫存、送出、退回更正、CSV）
- 帳號管理與登入紀錄查核

## 2. 啟動系統（本機）
1. 進入專案資料夾：`C:\Users\User\Playground\ISMS-Form-Redesign`
2. 使用任一靜態伺服器啟動（例如你已使用的 `.codex-local-server.cjs`）
3. 用瀏覽器開啟首頁（不要直接雙擊 `index.html`）

提示：若你在 `C:\WINDOWS\system32` 執行 Node 會找不到檔案，請先切到專案目錄再執行。

## 3. 登入與密碼
### 3.1 登入
1. 輸入帳號、密碼
2. 按「登入系統」
3. 登入成功後進入主畫面

### 3.2 忘記密碼
1. 登入頁按「忘記密碼？」
2. 輸入帳號綁定的電子郵件
3. 系統產生新密碼（畫面顯示）
4. 用新密碼重新登入

## 4. 角色與權限
- `最高管理員`：全功能（帳號管理、檢核表管理、教育訓練名單管理、退回更正、登入紀錄）
- `單位管理員`：可開立矯正單、填報檢核表、填報教育訓練
- `填報人`：可填教育訓練時數、可處理分派到自己的矯正單

## 5. 主選單功能
### 5.1 儀表板
- 看總件數、待矯正、逾期、本月結案
- 可快速點進最近矯正單

### 5.2 矯正單列表
- 可用關鍵字搜尋
- 可按狀態篩選
- 點一列可看詳情

### 5.3 開立矯正單（單位管理員以上）
1. 填寫基本資訊（提出單位、提出人、處理單位、處理人）
2. 填寫缺失分類、問題描述、期限
3. 送出後產生單號

系統規則：
- 提出單位會預設為登入者單位
- 處理人員會依「處理單位」自動過濾

### 5.4 矯正措施回填（被指派處理人或管理者）
1. 在矯正單詳情按「回填矯正措施」
2. 填矯正措施、根因、根因消除措施
3. 可上傳佐證（圖片/PDF）
4. 送出後狀態改為「已提案」

### 5.5 審核與追蹤（管理者）
- 已提案 → 可「進入審核」
- 審核中可選：
  - 通過結案
  - 轉追蹤
  - 退回重填
- 追蹤中可新增追蹤紀錄（最多 3 次）

### 5.6 內稽檢核表
### 填報
1. 進入「填報檢核表」
2. 逐題勾選符合狀態並填寫說明
3. 可「儲存草稿」或「送出檢核表」

### 草稿規則
- 草稿可續填
- 正式送出後不可直接修改

### 年度規則
- 自評日期變更時，稽核年度會自動換算（民國年）

### 5.7 教育訓練時數統計
### 一般填報人員
1. 進入「教育訓練時數統計」→「新增填報」
2. 依名單填各人時數與備註
3. 可新增名單外人員
4. 可「儲存暫存」
5. 正式送出前需上傳簽核掃描檔

### 管理者功能
- 可在「名單管理」匯入各單位名單
- 可刪除名單人員（一般填報人不可刪）
- 可查看全校填報進度與達標率圖表
- 可把正式送出的資料「退回更正」

### 匯出
- 可匯出全體統計 CSV
- 可匯出單一填報單 CSV

### 5.8 帳號管理（管理者）
- 新增使用者
- 編輯使用者
- 刪除使用者（`admin` 保留）

### 5.9 登入紀錄（管理者）
- 查看登入成功/失敗紀錄
- 系統保留最近 500 筆
- 可一鍵清除紀錄

## 6. 常見問題
### Q1. 點某個頁面後畫面不正常
- 先重新整理頁面
- 確認網址 hash 是否正確（例如 `#training`、`#checklist`）

### Q2. 為什麼我看不到某筆資料
- 系統會依角色與單位限制可見範圍
- 請確認你的帳號角色與單位設定

### Q3. 為什麼不能修改教育訓練單
- 狀態若是「正式送出」會鎖定
- 必須由管理者先退回更正

### Q4. 為什麼資料在另一台電腦看不到
- 目前前端版本資料存於瀏覽器 `localStorage`
- 換瀏覽器/電腦不會自動同步

## 7. 使用建議（管理者）
- 每月匯出 CSV 做校務留存
- 定期檢查登入紀錄
- 正式上線建議改為後端資料庫（例如你規劃的 Google Apps Script + Sheet）避免單機儲存風險
