# 教學文件：如何架設本地端伺服器並分享給網路上的人測試

當我們在開發前端網頁時，常常需要給主管、客戶或同事看目前的進度。這份文件會教您如何用最簡單的方式，在自己的電腦上啟動伺服器，並產生一個「測試用臨時網址」讓別人連線進來看！

---

## 步驟一：開啟本地端伺服器 (Local Server)

因為純前端專案有時會因為瀏覽器的安全性限制（CORS）導致直接雙擊打開 `index.html` 時無法正確載入檔案，所以我們需要開一個本地伺服器。

1. 在專案資料夾（`ISMS-Form-Redesign`）內點擊右鍵開啟終端機 (Terminal / PowerShell)。
2. 輸入以下指令啟動本機的 Caddy 伺服器，並指定跑在 8081 這個 Port 上：
   ```powershell
   .\caddy.exe file-server --listen :8081
   ```
3. 看到幾行字跳出來後，代表啟動完成了！現在您只要打開瀏覽器，網址輸入 `http://localhost:8081` 就可以看到您正在開發的網頁。

> **提示：** 只要這個視窗沒關掉，您修改程式碼並存檔後，重新整理瀏覽器網頁就會更新。

---

## 步驟二：建立對外的安全連線通道 (Tunnel)

只有 `localhost` 的網址，別人是連不進來的（因為那代表「您自己的電腦」）。我們需要一個對外曝光的安全通道。

1. **請保持剛才那個啟動伺服器的終端機視窗開啟**。
2. 開啟一個「**新的**」終端機視窗（確保您的路徑還是在專案資料夾內）。
3. 輸入以下這段指令，向 `localhost.run` 服務借用一個免費且隨機的安全網址：
   ```powershell
   ssh -o StrictHostKeyChecking=no -R 80:127.0.0.1:8081 nokey@localhost.run
   ```
4. 跑完之後，您會在畫面上看到幾行英文，其中會夾帶一條像這樣的網址：
   👉 `https://xxxxxxxxx.lhr.life`
5. 大功告成！您只要把這串附帶著 `https` 開頭的網址複製起來傳給別人，他們就可以從手機或他們的電腦上看到您的這個網頁了！

---

## 步驟三：關閉伺服器與通道 (結束分享)

這個外網分享的安全性非常高。因為它：
1. **網址是隨機亂數**：沒有人能用猜的猜中您的網址。
2. **存取權限極小**：對方只能看到您專案資料夾（`ISMS-Form-Redesign`）下的這幾支程式，無法看您電腦裡的其他機密資料。

當您測試完畢，想要關閉對外分享時，只要做以下動作之一：
- 點擊終端機視窗右上角的 **[ X ]** 把這兩個黑框框都關掉。
- 或是在終端機點一下，按下鍵盤的 `Ctrl + C` 中止程式。

一旦您這麼做，剛剛分享給別人的 `https://xxxxxxxxx.lhr.life` 網址就會**瞬間報廢**，再也沒有人可以連進來了，百分之百安全！

---

### 常見問題 (Q&A)

**Q: 對方跟我說他打開網址後畫面破圖或出不來？**
A: 因為我們使用的是免費的通道服務，有時候網路連線不穩定會導致 CSS 被卡住沒傳輸過去。這時候只要把第二步的通道終端機（ssh 那個）關掉並重新啟動一次，拿一個新的亂數網址給對方測試就可以了！
